## Summary

Minor release making occasions, materials, and seasons editable across the
product for the first time, adding a live `/status` page that replaces the
stale hardcoded About-page AI card, tightening similar-item detection with a
color-family gate, and polishing wear-log UX. Deterministic engines still
decide; AI never writes relations or interprets status
([ADR-005](../adr/ADR-005-ai-does-not-decide.md)).

## Product surfaces

- **Item form (create + edit)** ‚Äî chip multi-selects for Occasions, Materials,
  Seasons; deterministic **Suggest** button (keyword-matched concepts, never
  auto-saves)
- **Bulk actions** ‚Äî Add/Remove occasion and material
- **Item detail** ‚Äî Edit entry points on Style DNA and Occasions cards; item
  preview modal from wear-log detail
- **`/status`** ‚Äî live AI wiring (override badges), passive service health,
  manual Run checks probes, OpenAI budget card, build info
- **About** ‚Äî hardcoded AI Runtime card replaced with a `/status` link
- **Catalog Review** ‚Äî color-family gate removes cross-family similar-item
  false positives (Navy vs Grey Blazer)
