# Wardrobe OS v2.0.1 — Inventory Image Intelligence

**Release date:** 2026-07-12  
**Tag:** `v2.0.1`  
**Previous:** [v2.0.0](./v2.0.0.md)

## Summary

Patch release that connects **owned inventory primary photos** to StyleDNA and
RecommendationContext via user-confirmed visual attributes ([RFC-020](../rfc/RFC-020-Inventory-Image-Intelligence.md)).
Vision proposes; the owner Accepts or Rejects; engines still decide
([ADR-005](../adr/ADR-005-ai-does-not-decide.md)).

## Features

- **Analyze primary image** on item detail — reuses Vision Engine
  (`analyzeImageRequest` / `/api/ai/vision`); never duplicates perception.
- **`VisualStyleAttributes`** stored separately from manual wardrobe fields
  (`pending` | `accepted` | `rejected` | `stale`).
- **Accept / Reject** on item detail; Accept only enables StyleDNA gap-fill
  (manual colour / material / formality / tags always win).
- **Stale on primary change** — re-upload or set-primary marks attrs stale.
- **Developer backfill** at `/developer/inventory-images` — batch Analyze only;
  never auto-accept.
- **Recommendations** load accepted visuals and merge before `deriveStyleDNA`.

## Architecture Changes

| Layer | Change |
| --- | --- |
| Domain | `src/domain/inventory-image-intelligence/` — analyzer + StyleDNA merge (pure) |
| Repository | `visual-attributes.repository.ts` → `item_visual_attributes` |
| Service | `visual-attributes.service.ts` orchestrates Vision → persist → Accept/Reject |
| Hooks | React Query hooks in `src/features/inventory/hooks` |
| UI | `ItemVisualAnalysisCard` on item detail; `InventoryImageBackfillView` |
| Reco | `RecommendationContextBuilder` + recommendations service merge path |

## Schema Changes

Additive only — already applied on the Wardrobe Supabase project:

- Migration: [`docs/migrations/RFC-020-inventory-image-intelligence.sql`](../migrations/RFC-020-inventory-image-intelligence.sql)
- Table: `item_visual_attributes` (unique on `item_id`, RLS `mvp_anon_*`)
- Types: `types/database.ts` includes the new table

No columns added to `wardrobe_items`. Manual fields are never overwritten by
this table.

## Breaking Changes

None. Patch-compatible with v2.0.0.

## Migration Notes

1. Ensure `docs/migrations/RFC-020-inventory-image-intelligence.sql` has been
   applied (user confirmed applied for this cut).
2. Deploy app with `package.json` / tag `v2.0.1`.
3. Existing items stay unanalyzed until Analyze or Developer backfill.
4. Accepted attrs enrich recommendations only after Accept + confidence ≥ 0.5.

## Known Limitations

- Backfill is sequential and developer-gated; no background job queue.
- Low-confidence accepted attrs contribute nothing to merge (threshold 0.5).
- Visual attrs are one row per item (primary image); multi-image consensus is
  out of scope.
- Lint baseline from v2.0.0 remains (8 `react-hooks/set-state-in-effect`
  errors + 2 warnings) — not introduced by this release.
