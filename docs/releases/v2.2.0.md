# Wardrobe OS v2.2.0 — Catalog Review v2

**Release date:** 2026-07-12  
**Tag:** `v2.2.0`  
**Previous:** [v2.1.0](./v2.1.0.md)

## Summary

Minor release upgrading `/inventory/review` from name-only duplicate cleanup
into **Catalog Review**: metadata-aware duplicates, similar items, completeness
sections, dismissals, and a deterministic quality score. AI never decides
catalog correctness ([ADR-005](../adr/ADR-005-ai-does-not-decide.md)).

## Included RFCs

| RFC | Title | Status in v2.2.0 |
| --- | --- | --- |
| [RFC-024](../rfc/RFC-024-Catalog-Review-v2.md) | Catalog Review v2 | ✅ Shipped |

## Product surfaces

- **Catalog Review** — `/inventory/review` (renamed from Import Review)
- Sections: Duplicates, Similar Items, Missing Metadata, Unbranded, Missing
  Images, Visual Analysis Pending, Data Quality Issues
- Actions: dismiss / ignore, mark reviewed, edit, add image / analyze, retire /
  hard delete (confirmation preserved)

## Architecture Changes

| Layer | Change |
| --- | --- |
| Domain | `src/domain/catalog-review/` — engines + tests |
| Persistence | `catalog_review_dismissals`, `catalog_review_item_state` |
| Service / repo | Catalog review orchestration; cleanup modes unchanged |
| UI | Overview cards, tabs, filters, search, severity actions |

## Schema Changes

Additive migration (already applied on Wardrobe):

- [`docs/migrations/RFC-024-catalog-review-v2.sql`](../migrations/RFC-024-catalog-review-v2.sql)

## Breaking Changes

None. Minor-compatible with v2.1.x. Cleanup defaults unchanged (retire;
hard delete requires explicit opt-in).

## Migration Notes

1. Apply `RFC-024-catalog-review-v2.sql` if missing.
2. Deploy app with `package.json` / tag `v2.2.0`.
